# 公众号目录补充最新文章标题

## 背景

当前 `公众号 AI 解读汇总` 页面中的目录项仅显示：

- `公众号名（篇数）`
- `最新：发布时间`

这会让用户只能看到更新时间，无法直接判断该公众号最新一篇文章是什么，目录的扫描效率不够高。

## 目标

在不改变现有分组、排序、折叠和跳转行为的前提下，让目录项同时显示：

- 公众号名
- 篇数
- 最新文章发布时间
- 最新文章标题

## 方案

采用单行展示，目录项格式调整为：

`公众号名（篇数）｜最新：发布时间｜标题：最新文章标题`

如果最新文章标题为空，则回退为：

`标题：(无标题)`

如果最新发布时间为空，则继续保持当前行为，不展示时间片段，仅展示标题片段。

## 数据来源

目录项继续复用每个公众号分组后 `sorted_items[0]` 作为“最新文章”来源：

- 时间：沿用当前 `_format_latest_time()` 结果
- 标题：新增从最新 item 中读取 `title`

不新增新的存储字段，不修改 JSON 缓存结构。

## 实现范围

仅修改目录构建和渲染相关逻辑：

- `scripts/wechat_article_crawler/article_analysis.py`

补充对应回归测试，覆盖目录项标题展示：

- `tests/test_article_analysis.py`

## 非目标

- 不修改账号分组规则
- 不修改目录按分类展示规则
- 不修改正文卡片的标题和时间展示
- 不修改“历史解读”折叠逻辑
- 不修改重解读按钮逻辑

## 验证

- 运行聚焦测试，确认目录 HTML 包含最新标题文案
- 重新生成 `output/article_analysis/index.html`
- 用浏览器或 HTTP 方式确认目录文案正确渲染
